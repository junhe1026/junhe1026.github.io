.gitignore 文件修改后不生效的解决办法

心得
git 版本控制

参考这里:http://segmentfault.com/q/1010000000430426, 被采纳的回答。

在git 版本控制的时候，往往会碰到.gitignore文件记录不全的问题。尤其是在使用IDE开发时，IDE会产生很多隐藏缓存或者配置文件。当我们准备提交时，混入这些不想提交的缓存，我们通常会在.gitignore文件中继续添加ignore记录。然后我们git checkout这些记录，并提交。

当我们下次提交时，这些文件又阴魂不散出现在里面。其实，.gitignore文件中的记录只会去匹配还没有被track的文件，那些已经提交过的文件，即使你后来再加.gitignore，git 也不会去忽略它们。

正确的做法是：

  1. 清除这些不需要提交文件的track, 比如：我们不想要提交Pycharm的.idea配置文件夹
  1. 在.gitignore文件中添加这条记录
  2. 提交并推送

完成！ 以后就不会再出现这些文件了。

P.S.: GitHub 在创建新版本库时，可以选择自动生成一个开发语言相关的.gitignore文件，里面的记录还是蛮全的，我们自己在新建项目时，可以参考这个文件。
